# 开发核心规则（第一规则）

以下规则具有最高优先级，所有开发工作必须严格遵守：

## 1. 单版本原则

- 不允许创建多个版本的代码
- 只维护一个版本，所有修改都在第一版代码上进行
- 禁止重写、重构成新版本

## 2. 真实数据原则

- 不能使用模拟数据（mock data）
- 不能生成 demo 代码
- 所有代码都是生产级别的实现
- 必须使用真实数据集进行训练和测试

## 3. 代码组织原则

- 严格遵守开发流程
- 代码放在对应文件夹里
- 遵循 spec 中定义的目录结构

## 4. 禁止重复

- 不允许有重复功能的代码
- 每个功能只实现一次
- 复用已有代码，不要重新实现

## 5. 代码完整性原则（禁止简化）

- 禁止简化代码（No Code Simplification）
- 必须保持代码的完整性和复杂度
- 不允许为了"简洁"而删除必要的功能或逻辑
- 所有数值方法必须完整实现，不能省略关键步骤
- 验证代码必须包含完整的物理模型，不能使用简化近似

## 6. 代码完整性原则（禁止删除）

- 禁止简化已有代码
- 不允许删除或省略任何功能实现
- 修改代码时必须保持原有功能完整
- 不能用占位符、TODO 或注释替代实际代码
- 所有物理模型和数值算法必须完整实现，不可简化
